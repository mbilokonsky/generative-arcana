# Suits — Manual

Adopt a four-fold structure the theme already supplies. Best when forcing a dialectical cross-product would fight an obvious native grouping (four classical elements, four houses, four seasons, four nations, four humors, four directions).

## 1. Find the native four

Identify exactly four categories the theme treats as a natural, roughly-symmetric partition. Test: **exhaustive-ish** (together they cover the theme), **roughly symmetric** (peers, none swallows the others), **distinct** (each its own meaning palette and visual language). If the theme offers three or five, drop/merge with a stated rationale or fall back to `dialectical.md`.

## 2. Recover the latent dialectics (recommended)

Even native suits usually imply two crossed axes (elements imply hot⟷cold × wet⟷dry; seasons imply waxing⟷waning × warm⟷cold). Naming them sharpens meanings and keeps the four orthogonal — record them in each suit's `description`.

## 3. Order them and build

**Assign each suit an `index` (0–3)** — fixed and load-bearing (the transversal's minor walk reads it; the first suit's Ace is the origin station). If the theme has a canonical order (fire/water/air/earth, the seasons' cycle), use it; otherwise choose a stable order.

For each, the full `Suit` per `references/schema.md` — index, name, slug, glyph SVG (per `references/svg_symbols.md`, distinct at 16px), meaning palette (3–6 upright + 3–6 chiral inverted), `visual_style`. Decide intra-suit coherence deliberately.

Lean on the suits' existing cultural associations for the palettes, but still chiralize inversions (Fire inverted is not "no fire" but "consuming what feeds it").

Present the four ordered suits for feedback before Stage 2.
