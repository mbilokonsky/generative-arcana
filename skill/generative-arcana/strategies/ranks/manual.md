# Ranks — Manual

Adopt a rank progression the theme supplies. Best when manual suits were chosen (tonal consistency) or the theme has a native ten-stage process mapping cleanly onto Ace–Ten (ten phases of a ritual, ten levels, ten rungs).

Each rank is a `Rank` per `references/schema.md`: description, meaning palette (upright + chiral inverted), `visual_content`.

## Numbered ranks (Ace–Ten)

1. **Find the native ten.** A sequence of exactly ten ordered stages. Ace = origin/potential; Ten = completion/overflow/transition. A different count → stretch/compress with a rationale, or fall back to `questions.md`.
2. **Keep it suit-agnostic.** Each rank must refract through *all four* suits — write descriptions and `visual_content` as abstractions ("consolidation", "a structure under construction"), not suit-specific scenes. A rank that only fits one suit is a bug.
3. **Make the progression mean something.** Ace→Ten should encode a real ascent or cycle; state the logic in the descriptions.

The **fourth axis stays latent** (`references/numeric_axis.md`): even a theme-native progression has prime-numbered ranks that tend toward the irreducible and composite ranks that tend toward the derived. The integrator may lean on this; you needn't structure for it.

## Face ranks (11–14)

Build as in `ranks/questions.md`: four roles, distinct initials, no collisions with suit or major names, an encoded progression. If the theme supplies four native roles, use them — confirm distinct initials and adjust collisions.

Present the ten numbered ranks, then the four face ranks, for feedback.
